# ⚠️ 已知問題與設計遺漏

## v1.2.3 發現並修復的問題

### Bug #1: Mask Calculator 邊界條件問題

**嚴重程度**: ⚠️ 中等

**影響範圍**: 
- 當 `max_flash_addr == base_addr` 時（只有一個表格且位於 base）
- Mask 計算會返回錯誤的結果

**原始程式碼問題**:
```python
# 原始程式碼 (有 Bug)
def calculate_dynamic_mask_bytes(max_flash_addr, base_addr):
    if max_flash_addr <= base_addr:
        return 1, 0  # ← total_bits = 0 是錯誤的
    
    # ...
    total_bits = (offset // 256) + 1
    return max(1, mask_bytes), total_bits

# 問題在於 calculate_enable_mask_with_gaps:
for item in enabled_items:
    bit_pos = (addr - base_addr) // 256
    
    if 0 <= bit_pos < total_bits:  # ← 當 total_bits = 0 時，永遠為 False！
        mask_value |= (1 << bit_pos)
```

**問題情境**:
```
Base Address = 0x1000
只有一個表格，位址 = 0x1000

calculate_dynamic_mask_bytes(0x1000, 0x1000) → (1, 0)  # total_bits = 0

在 calculate_enable_mask_with_gaps 中:
  bit_pos = (0x1000 - 0x1000) // 256 = 0
  條件: 0 <= 0 < 0 → False ❌
  mask_value = 0  # 錯誤！應該是 1
```

**修復後的程式碼**:
```python
# 修復版本 (v1.2.3)
def calculate_dynamic_mask_bytes(max_flash_addr, base_addr):
    if max_flash_addr < base_addr:
        return 1, 1  # ← total_bits = 1，確保至少一個位元
    
    offset = max_flash_addr - base_addr
    total_bits = (offset // 256) + 1  # 確保至少為 1
    
    return max(1, mask_bytes), total_bits
```

**驗證**:
```
測試案例: Base=0x1000, 只有一個表格 addr=0x1000

修復前: mask = 0x00 ❌
修復後: mask = 0x01 ✅
```

**狀態**: ✅ 已在 v1.2.3 中修復

## 原始程式碼中可能存在的其他設計考量

### 設計考量: CRC 計算長度

**確認**: CRC-16 計算使用 **254 bytes** (0x00-0xFD)，而非完整的 256 bytes。

**原因**: 這是為了預留空間給 CRC 值：
- 位址 0xFE: CRC MSB
- 位址 0xFF: CRC LSB

```python
# 計算 CRC: 0x00 到 0xFD (254 bytes)
crc_val = calculate_custom_crc16(page_buffer[:254])

# 嵌入 CRC
page_buffer[254] = (crc_val >> 8) & 0xFF  # MSB
page_buffer[255] = crc_val & 0xFF            # LSB
```
